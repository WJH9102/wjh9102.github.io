# 快速开始指南

## 📖 本地预览

### 方法一：使用启动脚本（推荐）

```bash
./start.sh
```

然后访问 http://localhost:3000

### 方法二：手动启动

```bash
# 使用 docsify-cli
docsify serve .

# 或使用 Python
python3 -m http.server 3000
```

## 🚀 部署到 GitHub Pages

### 1. 推送代码

```bash
git add .
git commit -m "更新文档"
git push origin main
```

### 2. 启用 GitHub Pages

1. 进入 GitHub 仓库设置
2. 找到 "Pages" 选项
3. Source 选择 "main" 分支
4. 保存

### 3. 访问网站

https://wjh9102.github.io

## ✏️ 添加新文章

### 1. 创建 Markdown 文件

在对应目录下创建 `.md` 文件，例如：

```
烂笔头/新文章.md
```

### 2. 更新侧边栏

编辑 `_sidebar.md`，添加链接：

```markdown
* 📝 烂笔头
  * [ES 相关](烂笔头/ES%20相关.md)
  * [经验之谈（采坑日记）](烂笔头/经验之谈（采坑日记）.md)
  * [新文章](烂笔头/新文章.md)  <!-- 新增 -->
```

### 3. 添加图片

1. 将图片放在 `assets` 文件夹
2. 在文章中引用：

```markdown
![图片描述](assets/图片名称.png)
```

## 🎨 自定义配置

### 修改主题色

编辑 `index.html`，找到：

```css
:root {
  --theme-color: #42b983;  /* 修改这里 */
}
```

### 修改网站标题

编辑 `index.html`，修改两处：

```html
<title>我的个人笔记</title>
```

```javascript
window.$docsify = {
  name: '我的技术笔记',
}
```

### 更换主题

编辑 `index.html`，修改主题链接：

```html
<!-- 可选: vue.css, buble.css, dark.css, pure.css -->
<link rel="stylesheet" href="//cdn.jsdelivr.net/npm/docsify@4/lib/themes/vue.css">
```

## 💡 提示

1. **文件名空格处理**：URL 中空格会被编码为 `%20`
2. **图片路径**：使用相对路径，如 `assets/image.png`
3. **刷新缓存**：修改后按 `Ctrl+F5` 强制刷新
4. **Markdown 语法**：支持标准 Markdown 和扩展语法

## ❓ 常见问题

**Q: 本地预览正常，部署后样式错乱？**

A: 检查 `.nojekyll` 文件是否存在

**Q: 图片无法显示？**

A: 确保图片路径正确，使用相对路径

**Q: 搜索功能不工作？**

A: 清除浏览器缓存，重新加载页面

**Q: 如何修改页脚？**

A: 编辑 `index.html` 中的 `plugins` 部分
