# 多线程及并发原理

# 1.notify和notifyAll

当前线程获取锁后调用锁对象的 wait 方法会进入等待池，notify 会随机唤醒等待池中的一个线程让其进入锁池，而 notifyAll 会将等待池中的所有线程唤醒让其进入锁池去竞争锁。

# 2.yield方法

暗示线程调度器当前线程愿意让出 cpu 使用权，但是线程调度器可能会忽略。

# 3.interrupt方法

调用 interrupt 方法通知线程应该中断了

1. 如果线程处于被阻塞状态，那么线程将立即退出被阻塞状态，并抛出一个 InterruptedException 异常
2. 如果线程处于正常运行状态，那么会将线程的中断标志设置为 true 。被设置中断标志的线程将继续运行，不受影响。

# ReetrantLook和synchronized区别

1. ReetrantLook位于 JUC 包下
2. 基于 AQS 实现
3. 比 synchronized 更细粒度的控制，如公平性
4. 调用 lock() 后必须调用 unlook () 释放锁
5. 可重入，并不一定不 synchronized 效率高
