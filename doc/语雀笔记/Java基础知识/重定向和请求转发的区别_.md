# 重定向和请求转发的区别_

1. 请求转发（Forward）是服务器行为，重定向（ Redirect）是客户端行为。请求转发可以通过HttpServletRequest的 getRequestDispatcher()方法链调用forward()方法实现即`req.getRequestDispatcher("url").forward(req,res)`;重定向是利用服务器返回的状态码来实现的。客户端浏览器请求服务器的时候，服务器会返回一个状态码。服务器通过 HttpServletRequestResponse的setStatus(int status)方法设置状态码。如果服务器返回301 或者 302，则浏览器会到新的网址重新请求该资源。转发可以通过 HttpServletResponse 对 象的 sendRedirect()方法实现。
2. 地址栏显示：请求转发是服务器取请求资源，服务器直接访问目标地址的URL，将URL响应的内容读取然后返回给浏览器，浏览器根本不知道响应的内容来自哪里，所以地址栏还是原来的地址；重定向是服务端根据逻辑返回一个状态码，告诉浏览器取请求那个地址，然后浏览器去请求新的地址，所以浏览器地址栏显示的是新的URL
3. 数据共享：请求转发可以共享request域里面的数据，而重定向不能共享request域里面的数据
4. 应用场景： 请求转发只能访问当前WEB的应用程序，当需要访问其他WEB站点的资源时需要使用重定向
5. 效率：请求转发较高，重定向较低；在使用请求转发和重定向都无所谓时建议使用请求转发
