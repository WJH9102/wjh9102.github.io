# 补数说明

## 1.calculation 组件

为了不影响生产正常的数据消费流程，建议新部署一个 calculation 组件，并修改以下配置：

- kafka-config.yml

```yaml
spring:
  kafka:
    topics:
      # 新建一个 dics topic 区别于另一个 calculation
      dics: zhaoshang_test
      # 新建一个 ims topic 区别于另一个 calculation
      ims: zhaoshang_test2
```

- business-config.yml

```yaml
schedule:
  merge:
    # 新增的配置，区别于另一个 calculation(默认值是 merge_)
    queuePrefix: mergebak_
```

## 2.补数工具

补数工具的 kafka-config.yml 中的 topic 主题配置为新的 topic 即可
